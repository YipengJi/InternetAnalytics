Remove stopwords and punctuation¶

Why: Punctuation only helps humans to structure and read the sentences but does not carry useful information about the topic. Thus, we can remove it. Stopwords are similar, as they represent the most common words in a language and usually only help connect and build a sentnce without adding meaning to it (e.g. think of articles (a,the..) or connectors (and,or, hence..) ). So stopwords are also removed.

import nltk

from nltk.stem.porter import *

def preprocess(course_list):
    #removing stopwords, punctuation, splitting numbers and words
    

    #dict to count frequency
    word_freq = {}
    processed_courses= course_list.copy()

    for course_id, course in enumerate(courses):
        #preprocess the description
        description = course['description']
        
        
        wordlist = []
        for line in word_tokenize(description):
            line = line.lower() # to lowercase
            line = digit_split(line) #split words concatinated with a number (without space inbetween)
            
            for word in line:
                #remove stopwords and puctuation and check if word alphabetic (probably unneccessary)
                if (word not in stopwords) and word.isalpha(): 
                    wordlist.append(word)
                    #add to frequency list
                    word_freq[word]=word_freq.get(word,0)+1
                    
        processed_courses[course_id]['wordlist'] = wordlist
    
    return processed_courses,nltk.FreqDist(word_freq)

Remove the very frequent words¶

Why: Below we can see the word distribution of the 35 most frequent words over all documents (not cumulated). This is a power law distribution. That means that the most common words are used much more often than the other words. Using a word too often makes a word useless for topic classification as it will be part of any topic. The most frequent words (methods, learning, student) prove this point. Of course they are used often in course descriptions because the words are closesly connected to university but they are too broad to indicate a specific topic or field of studies. Therefore, we remove the most frequent words.

The plot below shows the most frequent words. We decide to delete the first ten words as methods..students..end seem very vague. After that the words assesment, outcomes, prerequisites seem very general,too but they might already contain some info. E.g. there might be some courses with more prerequisites than others.

def delete_words(course_list, to_delete):

    processed_courses= course_list.copy()

    for course_id, course in enumerate(course_list):

        for word in wordlist:
            #remove words to be deleted

Stemming vs Lemmatization¶

The goal of both stemming and lemmatization is to reduce inflectional forms and sometimes derivationally related forms of a word to a common base form.

Word stemming means removing affixes from words and return the root word. Lemmatization is a similar approach but based on a vocabulary and morphological analysis of words. It aims to return the base or dictionary form of a word (lemma).

So the difference is that stemming can create non-existent words, whereas lemmas are actual words.

Thus, lemmatization is probably more accurate as it works with knowledge of the language. We try both methods separately in order to be able to use either lemmas or stems for the later analysis and compare the results

In [12]:
#stemming an lemmatization
#https://likegeeks.com/nlp-tutorial-using-python-nltk/
#https://pythonprogramming.net/stemming-nltk-tutorial/?completed=/stop-words-nltk-tutorial/

from nltk.stem import PorterStemmer
from nltk.stem import WordNetLemmatizer

lm = WordNetLemmatizer()
ps = PorterStemmer()

def stemming(course_list,wordlist="cut_wordlist"):
    processed_courses= course_list.copy()
    all_stems = set()
    all_lems=set()

    for course_id, course in enumerate(processed_courses):
        words = course[wordlist]
    
        course_stems = []
        course_lems = []
        for word in words:
            course_stems.append(ps.stem(word))
            course_lems.append(lm.lemmatize(word))
            processed_courses[course_id]['stems'] = course_stems
            processed_courses[course_id]['lems'] = course_lems
            all_stems.update(course_stems)
            all_lems.update(course_lems)
    
    return processed_courses, all_stems,all_lems

Why: Even though each single word always provides some meaning, groups of words could give more information. That is because often concepts or ideas are not only described in one word. (e.g. social network). So we hope to win some precision by using bigrams on the lemmas.

def bigrams(course_list,wordlist="lems"):
    processed_courses= course_list.copy()
    all_bigrams = set()

    for course_id, course in enumerate(processed_courses):
        words = course[wordlist]
    
        course_bigrams = list(nltk.bigrams(words))
        processed_courses[course_id]['bigrams'] = course_bigrams
        
        all_bigrams.update(course_bigrams)
        
    
    return processed_courses, all_bigrams

from collections import Counter
from collections import OrderedDict
import  operator


import numpy as np
from scipy.sparse import csr_matrix

# compute tfidf and return the matrix and ordererd term list
# print highest terms of internet analysis description if needed
def tfidf_procedure(terms,term_name,prin=True):
    
    n = len(terms)
    m = len(processed_courses)
    TF_matrix = np.zeros((n, m))

    term_list=list(terms)
    term_list.sort()

    n_ids = term_list
    m_ids = [d['courseId'] for d in processed_courses]
    
    
    vals=[]
    cols=[]
    rows=[]
   
    #TF-matrix
    for course_id, course in enumerate(processed_courses):
        c_terms = course[term_name]
        c_term_counter = Counter(c_terms)
        number_terms = len(c_terms)
        for key, value in c_term_counter.items():
            #get index of the term
            n_id = n_ids.index(key)
            TF_matrix[n_id][course_id] = value
            
            rows.append(n_id)
            cols.append(course_id)
            vals.append(value)
            
    
    #IDF        
    denom = np.apply_along_axis(np.count_nonzero, 1, TF_matrix)
    IDF_vector = np.log(m / denom)
    TFIDF_matrix = TF_matrix * np.array([IDF_vector]).T
    
    
    #1.  Print the 15 terms in the description of the IX class with the highest TF-IDF scores.
    if prin:
        IX_index = m_ids.index('COM-308')
        #get the values for the course
        IX_TFIDF_col = TFIDF_matrix[:,IX_index]
        #add all values hiogher zero and connect to a word/lemma/stem
        IX_TFIDF_vals = {}
        for ind, val in enumerate(IX_TFIDF_col):
            if not val > 0.0:
                continue
            IX_TFIDF_vals[n_ids[ind]] = val

        IX_TFIDF_vals_ordered = OrderedDict(sorted(IX_TFIDF_vals.items(),key = operator.itemgetter(1),reverse = True))
        for item in list(IX_TFIDF_vals_ordered.items())[:15]:
            print (item)
        
    
    return TFIDF_matrix, term_list, n_ids

def cosine_sim(d1, d2):
    return np.dot(d1, d2) / (np.linalg.norm(d1) * np.linalg.norm(d2))

2. What do you think of the results? Give your intuition on what is happening.¶

We observe that while the first query gives good results, the second query 'facebook' only returns one document with a non-zero 'similarity'. That is because the word 'facebook' does only occur in that one course description but not in others. So we are not be able to give any relevance for other documents, even though they might contain similar phrases like 'social media'.

This shows a disadvantage of vector space retrieval models, as they are not able to generalize the concepts of the searched terms and fail if the specific word cannot be found oin a document.
